GitHub Pagesってなに？

GitHub Pagesは静的なサイトをホスティングするサービスです。 HTML, CSS, JavaScriptなど。 GitHub のワークフローなどを用いてサイトを作成することができます。 必要に合わせてカスタムのドメインを当てることもできます。
GitHub Pagesのサンプル https://github.com/collections/github-pages-examples
GitHub Pagesでカスタムドメインを使う https://help.github.com/ja/articles/using-a-custom-domain-with-github-pages
GitHub Pages についてhttps://help.github.com/ja/github/working-with-github-pages/about-github-pages

GitHub Pages の使用料金

GitHub アカウントがあれば無料で作成できます。制約はあるので公式ページのHelpから「GitHub Pages について」をよく熟読ください。

GitHub Pages 注意点・警告

Warning: GitHub Pages sites are publicly available on the internet, even if their repositories are private. If you have sensitive data in your site's repository, you may want to remove it before publishing.

警告: GitHub Pages サイトは、リポジトリがプライベートであっても、インターネット上で公開されています。サイトのリポジトリに機密データがある場合は、公開する前に削除することを予めおこなってください。

GitHub Pages でページをつくってみましょう

1. GitHub に登録してなければアカウントを作りましょう。

2. GitHub Pagesを作りましょう https://help.github.com/ja/github/working-with-github-pages/creating-a-github-pages-site
GitHub でプロジェクトを作ります。この時Public:公開状態で作る必要があります。

作ったプロジェクトをcloneする
git clone https://github.com/{{アカウント名}}/{{プロジェクト名}}
cloneしてきたプロジェクトのディレクトリに移動する
cd {{プロジェクト名}}
index.html に内容を書き込む
echo "Hello World" > index.html
Gitリポジトリに含める
git add index.html
Gitリポジトリにコミットする
git commit -a -m "{{コミットメッセージ}}"
GitHubリポジトリにプッシュする : gh-pages ブランチにプッシュすることになります
git push origin gh-pages
作成コード全体
git clone https://github.com/{{アカウント名}}/{{プロジェクト名}}
cd {{プロジェクト名}}
echo "Hello World" > index.html
git add index.html
git commit -a -m "{{コミットメッセージ}}"
git push origin gh-pages

3. GitHub Pagesを更新していく

新たなファイルがある場合はGitリポジトリに含める
git add {{新たなファイルの名前:新規公開物ある場合}}
Gitリポジトリにコミットする
git commit -a -m "{{コミットメッセージ}}"
GitHubリポジトリにプッシュする : gh-pages ブランチにプッシュ
git push origin gh-pages
作業コード全体
git add {{新たなファイルの名前:新規公開物ある場合}}
git commit -a -m "{{コミットメッセージ}}"
git push origin gh-pages
